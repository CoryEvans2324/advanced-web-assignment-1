XML Syntax Rules

XML Documents Must Have a Root Element

In this example child is the root element.

< child >
< name > Billy < name >
< age > 7 < age >
< /child >

All XML Elements Must Have a Closing Tag

< message > There is both a opening and closing tag. < /message >

XML Tags are Case Sensitive

< MesSage > These tags are valid, but they must match exactly. < /MesSage >

XML Elements Must be Properly Nested

< message > < i > This is valid XML. < /i > < /message >
< message > < i > This is invalid XML. < /message > < /i >

XML Attribute Values Must Always be Quoted

< diaryEntry date="28/10/2021" >
< content > Dear diary... < /content >
< /diaryEntry >

JSON Syntax Rules

Data is in key/value pairs.

"name": "Cory"

Data is seperated by commas.

"name": "Cory", "age": 20

Curly braces hold objects.

{ "name": "Cory", "age": 20 }

Square brackets hold arrays.

[ 1, 2, 3, 4, 5 ]

An example:

{
"name": "Cory",
"age": 20,
"pets": [
{
"name": "Storm",
"type": "cat"
},
{
"name": "Zoe",
"type": "cat"
}
]
}
